Team Project

Engine

핵심 시스템

  • 농사 시스템: 작물 성장 주기(씨앗→새싹→성장→수확) + 수확물의 전투 연동(포션/강화)

  • 기술 구성: NavMesh Plus(2D 경로 탐색), FSM(상태 분리), 서버 연동 설계(ASP.NET Core)

Generative AI 활용

“생성형 AI 활용 개발 실무”에서 학습한 방법을 개발 흐름에 연결했습니다.

개발에 적용한 방식

  • 기능 명세를 먼저 구조화(입력/출력/예외 케이스) → 구현 난이도 감소
  • 코드 초안 생성 → 유니티 규칙(컴포넌트, 직렬화, 성능)에 맞춰 리팩토링
  • 에러 로그를 기반으로 재현 조건/원인 후보/수정 체크리스트 정리
  • 문서·발표자료는 초안 생성 후 스크린샷/근거로 보강

내가 지키는 품질 규칙

  • AI 결과는 “정답”이 아니라 “초안”으로 사용
  • 프로젝트 구조/네이밍/의존성은 사람이 통제
  • 동작 확인(Play Mode) + 로그/프로파일링으로 검증
  • 작게 만들고(프로토타입), 빠르게 개선(반복)

GitHub Pages 배포 방법

1) 파일 업로드

  1. index.html을 Repo 루트에 업로드
  2. assets/ 폴더도 같이 업로드(지금은 비어 있어도 OK)
  3. 커밋 후 main 브랜치에 반영

2) Pages 설정

  1. GitHub → Repo → Settings → Pages
  2. Source: Deploy from a branch
  3. Branch: main / (root)
  4. Save 후 생성된 URL로 접속

주의사항(깨지는 것 방지)

  • 이미지/파일 경로는 절대경로(/assets) 대신 상대경로(./assets) 권장
  • 리포지토리 Pages는 보통 https://<USER>.github.io/<REPO>/ 형태
  • YouTube 임베드는 네트워크 환경에 따라 차단될 수 있어 “영상 링크”도 함께 두는 게 안전
